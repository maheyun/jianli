## 1. 准备工作

### 1.1 导出数据

```bash
cd d:\简历\可视化
python export_for_powerbi.py
```

运行后，CSV 文件生成在 `可视化/powerbi/data/统一驾驶舱/`：

```
统一驾驶舱/
├── kpi_overview.csv              ← 月度 KPI 总览（GMV/复购率/ROI）
├── drill_repurchase.csv          ← 复购率 → 用户分层下钻
├── drill_roi_platform.csv        ← ROI → 平台下钻
├── cross_repurchase_highvalue.csv ← 复购率 → 高价值客户 → 品类×渠道交叉
├── cross_roi_jd_marginal.csv     ← ROI → 京东 → 边际 ROI 曲线
└── anomalies.csv                 ← 异常标记表
```

### 1.2 导入到 Power BI

1. 打开 **Power BI Desktop**
2. **获取数据** → **文本/CSV**
3. 选择 `统一驾驶舱/` 下的全部 6 个 CSV 文件，逐个导入
4. Power BI 会自动检测字段类型，确认日期列被识别为「日期」类型

### 1.3 关键：日期表

Power BI 的时间智能函数依赖日期表。在「建模」→「新建表」中：

```dax
日期表 = 
ADDCOLUMNS(
    CALENDAR(DATE(2025,1,1), DATE(2025,12,31)),
    "年", YEAR([Date]),
    "月", FORMAT([Date], "YYYY-MM"),
    "月名", FORMAT([Date], "MMMM"),
    "季度", "Q" & FORMAT([Date], "Q"),
    "星期", WEEKDAY([Date], 2),
    "是否周末", IF(WEEKDAY([Date], 2) > 5, "周末", "工作日")
)
```

将该表标记为日期表（右键 → 「标记为日期表」）。

### 2.2 KPI 卡片

#### GMV 卡片

创建度量值：

```dax
GMV = SUM('kpi_overview'[GMV])
GMV_PrevMonth = 
    CALCULATE([GMV], 
        DATEADD('日期表'[Date], -1, MONTH))
GMV_Change = 
    DIVIDE([GMV] - [GMV_PrevMonth], [GMV_PrevMonth])
```

在画布上添加「多行卡片」视觉对象：
- 字段 → `[GMV]`
- 格式 → 数据标签：显示单位「万」
- 条件格式 → 背景色：基于 `[GMV_Change]` 的值（正数绿色，负数红色）

#### 复购率卡片

```dax
复购率 = 
    DIVIDE(
        SUM('kpi_overview'[复购率]),
        COUNTROWS('kpi_overview'),
        0
    )
复购率_上月 = 
    CALCULATE(
        [复购率],
        DATEADD('日期表'[Date], -1, MONTH)
    )
复购率_变化 = [复购率] - [复购率_上月]
```

#### ROI 卡片

```dax
全域ROI = SUM('kpi_overview'[ROI])
```

## 4. 第三层：交叉归因 + 策略建议（钻取页）

创建第三页，命名为「🎯 交叉归因」。此页面也为隐藏页。

### 4.1 复购率 → 高价值客户 → 品类×渠道

使用 `cross_repurchase_highvalue`。

**图表：上周 vs 本周（按品类和渠道分组）**

```dax
交叉变化 = SUM('cross_repurchase_highvalue'[变化])
上周复购交叉 = SUM('cross_repurchase_highvalue'[上周复购率])
本周复购交叉 = SUM('cross_repurchase_highvalue'[本周复购率])
```

使用两个并排的簇状柱形图：
- 左图：「按品类」，轴 = `维度值`，筛选 `维度类型 = "按品类"`
- 右图：「按渠道」，轴 = `维度值`，筛选 `维度类型 = "按渠道"`
- 均为上周 vs 本周对比

**策略建议框**（使用文本框或卡片）：

```
🔬 根因分析：
交叉结论：高价值客户在「运动袜」品类（-14%）和「抖音」渠道（-11%）
的复购率下降最严重。

推测原因：抖音近两月加大投放拉新，引入大量低价引流用户，
这些用户首单购买袜子但缺乏品牌忠诚度，稀释了高价值客户的袜子复购数据。

💡 策略建议：
1. 收紧抖音投放人群包：排除低购买力标签
2. 袜子复购活动迁移天猫（老客集中地）
3. 抖音改为推外套/裤子（高客单价品），从源头筛选
4. 监控未来两周数据，复购率应在 2-3 周内回升
```

### 4.2 ROI → 京东 → 边际分析

使用 `cross_roi_jd_marginal`。

```dax
平均ROI_京东 = SUM('cross_roi_jd_marginal'[平均ROI])
边际ROI_京东 = SUM('cross_roi_jd_marginal'[边际ROI])
```

**折线图：边际收益递减**
- **X 轴**：`预算_万元`
- **Y 轴**：`[边际ROI_京东]`、`[平均ROI_京东]`
- 添加恒线：`Y = 1.5`（边际 ROI 警戒线）
- 添加垂直线：`X = 120`（当前预算位置）

**策略建议框**：

```
🔬 根因分析：
京东当前预算约 120 万，此时边际 ROI 已降至 ~1.5×。
每多投 1 元只赚 1.5 元——扣除商品成本和运营费用后基本不赚钱。
对比得物：预算仅 25 万，边际 ROI 仍有 ~8×。

💡 策略建议：
1. 京东预算从 22% 降至 10%（~55 万）
2. 释放的 65 万：40 万给得物，25 万给抖音
3. 预估全域 ROI 可提升 12-15%
```

## 5. 设置钻取联动

这是三层看板最核心的交互。

### 5.1 页面钻取

Power BI 原生支持钻取，无需写代码：

1. **第二层页面** → 页面信息 → 「钻取」→ 打开「钻取」
2. 添加钻取字段：`Drill_Category` 或具体指标名
3. 第一层图表右键 → 「钻取」→ 选择第二层页面

### 5.2 为图表添加钻取按钮

在第一层的 KPI 卡片所在区域添加透明按钮或使用 Power BI 的「钻取」右键菜单。

**推荐做法**：使用书签 + 按钮模拟钻取（比原生钻取更灵活）：

1. 为每个指标创建书签（第一层 → 第二层对应图表）
2. 在第一层 KPI 卡片上覆盖透明按钮
3. 按钮操作 → 「书签」→ 跳转到第二层对应书签

### 5.3 跨页面筛选同步

在第二层图表的「可视化」→「格式」→「编辑交互」中：
- 设置图表之间的交叉筛选关系
- 点击「高价值客户」柱 → 右侧图表自动筛选到该分层

## 6. 条件格式与异常标红

### 6.1 KPI 卡片条件格式

选中 KPI 卡片 → 「格式」→ 「条件格式」→ 「背景色」：

**规则（基于变化值）：**

| 条件 | 颜色 | 含义 |
|------|------|------|
| 变化 < -5% | 🔴 `#FEE2E2` | 严重下降 |
| -5% ≤ 变化 < -2% | 🟡 `#FEF3C7` | 轻微下降 |
| -2% ≤ 变化 ≤ +2% | ⚪ `#F1F5F9` | 正常 |
| 变化 > +2% | 🟢 `#DCFCE7` | 上升（利好） |

对于 CPA 这类「越低越好」的指标，规则反过来。

### 6.2 异常自动标红（DAX 度量值）

```dax
复购率_告警 = 
VAR change = [复购率_变化]
RETURN
    IF(change < -3, "🔴 告警：复购率下降超3%",
       IF(change < -1, "🟡 关注：复购率轻微下降",
          "🟢 正常"))
```

将这个度量值用在卡片或表中显示告警文字。

### 6.3 趋势线标记

在折线图上添加参考线：

```dax
GMV_30日均线 = 
CALCULATE(
    [GMV],
    DATESINPERIOD('日期表'[Date], LASTDATE('日期表'[Date]), -30, DAY)
) / 30
```

## 8. 发布与定时刷新

### 8.1 发布到 Power BI Service

1. **文件** → **发布** → **Power BI**
2. 选择工作区
3. 发布成功后，在浏览器中打开

### 8.2 设置定时刷新

1. 在 Power BI Service 中打开数据集
2. **设置** → **计划刷新**
3. 添加数据源凭据（本地文件需使用 **Power BI Gateway**）
4. 设置刷新频率：每天 8:00 AM

> **注意**：CSV 文件需要先更新。可以把 `export_for_powerbi.py` 加入 Windows 计划任务，每天早上 7:00 运行，Power BI 在 8:00 刷新。

### 8.3 数据更新流程

```
每天早上 7:00
  └─→ Windows 计划任务运行 export_for_powerbi.py
       └─→ 更新 powerbi/data/ 下的 CSV 文件
  
每天早上 8:00
  └─→ Power BI Service 自动刷新数据集
       └─→ 看板显示最新数据
```

## 附录：完整 DAX 度量值清单

### KPI 卡片类

```dax
// GMV
GMV_Total = SUM('kpi_overview'[GMV])
GMV_PrevWeek = CALCULATE([GMV_Total], DATEADD('日期表'[Date], -7, DAY))
GMV_WoW = DIVIDE([GMV_Total] - [GMV_PrevWeek], [GMV_PrevWeek])

// 复购率
复购率_Total = AVERAGE('kpi_overview'[复购率])
复购率_Change = [复购率_Total] - CALCULATE([复购率_Total], DATEADD('日期表'[Date], -30, DAY))

// ROI
ROI_Global = AVERAGE('kpi_overview'[ROI])
ROI_Change = [ROI_Global] - CALCULATE([ROI_Global], DATEADD('日期表'[Date], -30, DAY))

// 变化箭头
变化箭头 = 
VAR chg = [GMV_WoW]
RETURN IF(chg > 0, "↑", IF(chg < 0, "↓", "→"))

// 告警颜色
告警色 = 
VAR chg = [ROI_Change]
RETURN IF(chg < -0.05, "#EF4444", IF(chg < -0.02, "#EAB308", "#22C55E"))
```

### 下钻分析类

```dax
// 复购率变化（按分层）
分层复购变化 = SUM('drill_repurchase'[变化])

// 平台 ROI
平台ROI = SUM('drill_roi_platform'[ROI])
平台预算占比 = SUM('drill_roi_platform'[预算占比])

// 交叉分析变化
交叉维度变化 = SUM('cross_repurchase_highvalue'[变化])

// 边际 ROI
边际ROI = SUM('cross_roi_jd_marginal'[边际ROI])
平均ROI = SUM('cross_roi_jd_marginal'[平均ROI])
```

### 格式化类

```dax
// 格式化 GMV（万）
GMV_Display = FORMAT([GMV_Total] / 10000, "¥#,##0万")

// 格式化变化率
Change_Display = 
VAR arrow = IF([GMV_WoW] > 0, "↑", "↓")
VAR pct = FORMAT(ABS([GMV_WoW] * 100), "0.0%")
RETURN arrow & " " & pct
```
