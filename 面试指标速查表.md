## 🏪 跃动体育 · 店铺背景（面试自我介绍用）

> 一家百人规模的 DTC 运动服饰品牌，主营**运动袜（¥19.9~69.9）** 和**运动服（¥89~499）**，
> 天猫 / 京东 / 抖音 / 得物四个平台同时运营，已积累 **2 万注册用户**。

| 维度 | 数据 |
|------|------|
| 注册用户 | 20,000 人（项目一）+ 12,000 老用户（项目二） |
| 运营平台 | 天猫 38%、京东 25%、抖音 22%、得物 15% |
| 核心品类 | 运动袜（60 SKU，成本率 ~42%）、运动服（60 SKU，成本率 ~38%） |
| 用户行为记录 | ~28,000 条浏览/加购/下单/支付行为 |
| 历史订单 | 1,851 单（项目一），老用户人均 8.3 单（项目二） |
| 广告投放 | 1,284 条投放记录，覆盖四个平台 |

> **一句话介绍**："我做的是一个运动服饰 DTC 品牌的全链路数据分析，从日常运营看板、用户分层（RFM-G）、广告 ROI、到产品健康度，四个项目覆盖了电商分析的核心环节。"

## 一、流量指标

### PV（Page View）页面浏览量
```
PV = 所有页面被访问的总次数（每次加载/刷新 +1）
```
| 低 PV | 正常 | 高 PV |
|--------|------|-------|
| 流量没做起来 | 业务健康 | 流量规模大 |

> 🏪 **店铺实况**：项目一累计埋点数据 **~28,000 条**，日均 PV **~930**。用户行为表记录了 view / add_to_cart / place_order / pay 四种行为。
>
> **面试话术**：PV 单独看没意义，必须配合 UV。PV 大 UV 小 = 用户粘性好；PV ≈ UV = 用户来了就走。

### UV（Unique Visitor）独立访客
```
UV = 去重后的访客数（按 user_id / cookie 去重）
```
> 🏪 **店铺实况**：日均 UV **~495**，去重后有效付费用户 **744 人**（项目一）。
>
> **面试话术**：UV 衡量流量广度，PV/UV 衡量流量深度。我的看板里把两者做了 30 天趋势图，可以快速发现流量异常。

### PV/UV 比值（人均浏览深度）
| < 2 | 2~5 | 5~10 | 10+ |
|------|------|------|-----|
| 用户来了就走 | 正常 | 粘性好 | 极高粘性（警惕刷量）|

> 🏪 **店铺实况**：日均 PV/UV ≈ **1.9**，属于偏低水平——和店铺处于成长期、SKU 较少有关。

## 二、交易指标

### GMV（Gross Merchandise Volume）总商品交易额
```
GMV = SUM(有效订单金额)  — 通常排除 unpaid 和 cancelled
```
| 口径 | 含义 | 谁用 |
|------|------|------|
| 下单 GMV | 含未付款 | 前台/对外宣传 |
| **支付 GMV** | 仅已付款 | **分析师常用** |
| 确认收货 GMV | 仅确认收货 | 财务核算 |

> 🏪 **店铺实况**：我在 SQL 中用 `WHERE order_status IN ('paid','shipped','completed')` 筛选有效订单。项目一中支付 GMV 由 1,851 单贡献；老用户（项目二）人均消费金额 **¥4,594**，中位数 **¥3,251**，说明存在高消费用户拉高均值。

> **面试话术**：GMV 是"虚荣指标"，还必须看净收入 = GMV - 退款 - 折扣。我项目里只用 paid/shipped/completed 状态，确保数据干净。

### 客单价 AOV（Average Order Value）
```
AOV = GMV / 有效订单数
```
> 🏪 **店铺实况**：订单金额呈对数正态分布，均值约 **¥135**。老用户中"高价值深耕用户"平均消费 ¥9,076，是"流失风险用户"（¥1,184）的 **7.7 倍**，客单价差异本质上是品类拓展带来的——买了服装的用户客单价自然比只买袜子的高。
>
> **面试话术**：客单价提升三板斧——满减凑单、捆绑销售（袜子+服装套餐）、加购弹窗。我在项目二里对高潜用户推品类拓展套餐，预期客单价提升 12%。

### ARPU / ARPPU
```
ARPU  = GMV / 全部用户（含没买的）
ARPPU = GMV / 仅付费用户
```
> 🏪 **店铺实况**：项目一 20,000 注册用户中付费用户 744 人，付费渗透率仅 3.7%。ARPU 低但 ARPPU 高，说明"不是用户没钱，是大部分用户还没转化"——增长重点在激活而非提价。
>
> ARPU 看整体变现能力，ARPPU 看付费用户质量。

### 笔单价
```
笔单价 = GMV / 订单数（注意：与客单价相同概念）
```
> 有些公司叫法不同，面试时说清楚自己理解就行。

## 三、转化指标（面试重点，必考）

### 转化率 CVR（Conversion Rate）
```
整体转化率 = 付费用户数 / UV
```
> 🏪 **店铺实况**：日均付费率 **~8.95%**，波动范围 5.27%~13.23%。注意这个转化率的分母是"有行为记录的 UV"，不是"所有注册用户"——口径不同，数值差异很大。
>
> **面试话术**：转化率看"流量质量"。转化率跌了 → 要么流量不精准（投错渠道），要么落地页有问题。我项目一里从用户行为表计算了完整漏斗。

### 漏斗转化（Funnel）— 逐环节拆解
```
浏览 ──→ 加购 ──→ 下单 ──→ 支付
  ↓        ↓        ↓        ↓
 100%   加购率%   下单率%   支付率%

加购率  = 加购人数 / 浏览人数     ← 商品吸引力
下单率  = 下单人数 / 加购人数     ← 价格/运费是否劝退
支付率  = 支付人数 / 下单人数     ← 支付流程顺畅度
整体转化 = 支付人数 / 浏览人数     ← 四环节连乘
```

> 🏪 **店铺实况**：漏斗各环节转化率 — 浏览→加购 **86.5%**，加购→下单 **86.5%**，下单→支付 **95.8%**。加购率和下单率都很高（说明商品吸引力强），支付环节流失最少。整体漏斗转化率约 **71.7%**（浏览→支付），这个数字偏高是因为模拟数据中用户行为连续性好。

> **面试必问**：转化率突然下跌怎么排查？→ 逐层拆漏斗，看哪个环节出问题。加购率正常但下单率跌 = 竞品可能在搞促销。

### CTR（Click Through Rate）点击率
```
CTR = 点击数 / 曝光数
```
> 衡量素材/文案质量。CTR 高但 CVR 低 = 素材诱导性强但落地页差（"标题党"）。

## 四、广告投放指标（项目三重点）

> 🏪 **项目三背景**：我模拟了四个平台（天猫/京东/抖音/得物）共 **1,284 条**投放记录，计算了各平台的 ROI、CPC、CPA、CPM，并用四象限分析（CPC × ROI）指导预算重分配。

### ROI / ROAS
```
ROI  = (收入 - 成本) / 投入成本    ← 严格的投资回报
ROAS = 广告收入 / 广告花费         ← 广告圈通用"ROI"
```
| ROI 值 | 含义 |
|--------|------|
| < 1 | 亏钱，停投或优化 |
| = 1 | 不赚不赔 |
| 1~3 | 正常水平 |
| 3+ | 优秀 |

> 🏪 **店铺实况**：我计算了四个平台 ROI。用 ROI=1 做盈亏线，把各平台分到四象限中——高 ROI 低 CPC 是"低成本引流"区（加大预算）、高 ROI 高 CPC 是"高效拉新场"（维持投入）、低 ROI 低 CPC 需优化素材、低 ROI 高 CPC 考虑停投。

> **面试话术**：很多公司说的 ROI 其实是 ROAS（只看收入不看商品成本）。面试时我会主动区分，显出专业性。我还做了**归因分析**——对比末次点击归因和全域归因，发现抖音在全域归因下的贡献被传统模型低估了。

### CPC（Cost Per Click）单次点击成本
```
CPC = 广告花费 / 点击数
```
> 🏪 **店铺实况**：CPC 和 CVR 是跷跷板——抖音 CPC 可能最低但用户冲动消费多、退货率高；天猫 CPC 高但用户购买意图强、CVR 也高。我只投 CPC 低的渠道反而不划算。

### CPA（Cost Per Acquisition）单次获客成本
```
CPA = 广告花费 / 转化数（下单或注册）
```
> 🏪 **店铺实况**：全域平均 CPA 是所有指标中 CEO 最关注的——"拉一个付费用户花了多少钱？" 必须和 LTV 对比：LTV / CPA > 3 才健康。

### CPM（Cost Per Mille）千次展示成本
```
CPM = 广告花费 / 展示次数 × 1000
```
> 跟媒体平台有关，抖音 CPM ≈ 15-50 元，微信 ≈ 30-80 元。贵不贵取决于目标人群的精准度。

### 四指标关系速记
```
CPM → 看曝光成本（平台决定）
CPC → 看素材质量（CTR = CPC / CPM）
CPA → 看转化效率（CPA = CPC / CVR）
ROI → 看最终效果（投产比）
```

### 归因模型对比（加分亮点）
| 模型 | 逻辑 | 缺点 |
|------|------|------|
| **末次点击** | 转化功劳全算在最后一个点击渠道 | 忽略前面的种草环节 |
| **全域归因** | 按转化路径上所有触点分配功劳 | 计算复杂 |

> 🏪 我在项目三中对比了两种归因模型，发现抖音在末次点击模型下被高估（用户往往最后在抖音看到广告就下单），而天猫/京东在前期的浏览种草作用被低估了。这直接影响预算分配决策。

## 五、商品指标（项目四重点）

> 🏪 **项目四背景**：我分析了 **150 个产品**（运动袜 38 + 运动T恤 38 + 运动裤 38 + 运动外套 36），在天猫/京东/抖音/得物四个渠道售卖，累计 **340 万件**销量、总销售额 **¥6,680 万**。数据含每日销售明细和库存，支持趋势预测和补货建模。

### 售罄率（Sell-through Rate）
```
售罄率 = 已售数量 / (已售 + 当前库存) → 0~100%
```
| < 15% | 15%~40% | 40%~80% | 80%+ |
|-------|---------|---------|------|
| 滞销，清仓 | 一般 | 正常 | 爆款，快补货 |

> 🏪 **店铺实况**：150 个产品中 **78% 为爆款**（售罄率 > 55% 且毛利率 > 35%），畅销款 11.3%，一般款 10.7%。售罄率整体较高是因为模拟数据中新品大部分处于生命周期早期。实际工作中我会特别关注那 10.7% 一般款——它们正在滑向滞销的边缘。
>
> **面试话术**：售罄率只看"量"，配合毛利率看"质"。我的项目四用售罄率 × 毛利率做四象限产品分级，同时结合线性回归预测销量趋势，给出智能补货建议。

### 毛利率（Gross Margin）
```
毛利率 = (销售额 - 进货成本) / 销售额
```
> 🏪 **店铺实况**：四个品类的成本率不同——运动袜 ~42%（毛利率 ~58%）、运动T恤 ~38%（毛利率 ~62%）、运动裤 ~35%（毛利率 ~65%）、运动外套 ~32%（毛利率 ~68%）。品类越高端毛利率越高，但销量也越小——我在清仓策略中针对不同品类做了差异化定价。
>
> **面试话术**：毛利是企业的"造血能力"。不同品类毛利率差异大——服装通常 60-70%，电子产品可能只有 10-15%。我的分析中结合品类做差异化清仓策略。

### 四象限决策矩阵
```
            高毛利率
               ↑
   高利低量    |    明星产品
  (提升曝光)   |   (加大投入)
               |
  ─────────────┼────────────→ 高售罄率
               |
   问题产品    |    高量低利
  (清仓处理)   |   (优化成本)
               |
            低毛利率
```

> 🏪 **决策实例**：售价 ¥499 的外套毛利率 68% 但售罄率只有 25% → 落入"高利低量"象限 → 策略是提升曝光（加大天猫推荐位），而非降价。

### 库存周转天数
```
库存周转天数 = 平均库存金额 / 日均销售成本
```
> 🏪 **店铺实况**：预期优化后缩短 5 天。库存周转直接关系到现金流——同样 100 万库存，周转 30 天 vs 60 天，占用资金差一倍。

### 动销率
```
动销率 = 有销量的 SKU 数 / 总 SKU 数
```

### 平台销售分布（项目四真实数据）
| 平台 | 销量占比 | 销售额占比 | 特点 |
|------|---------|-----------|------|
| **天猫** | 38.0% | 37.9% | 最大渠道，综合型 |
| **京东** | 25.0% | 25.0% | 第二大渠道 |
| **抖音** | 22.0% | 22.0% | 增速最快 |
| **得物** | 15.0% | 15.0% | 潮流垂直渠道 |

> 🏪 双十一当天销量 **6.5 万件**（¥1,280 万），是平时的 **15 倍**；双十二 **5.5 万件**（¥1,070 万）。大促期间售罄率会猛拉，但毛利率因为打折会大幅下降——不能只看销量涨了就高兴。

### 补货模型（项目四亮点）
```
补货点 = 预测日销量 × 提前期 + 安全库存
建议补货量 = 预测日销量 × (提前期 + 7天缓冲) - 当前库存
```
> 🏪 我用**线性回归**对每个产品拟合日销量趋势（上升/平稳/下降），而不是简单用历史均值，这避免了"对衰退产品继续补货"的坑。

## 六、用户指标（项目二重点）

> 🏪 **项目二背景**：我分析了 **12,000 名注册超半年的老用户**，用 RFM-G 模型做了分层。用户行为跨度 1~3 年，涉及袜子/服装两个品类，核心问题是"老用户 GMV 占比持续下滑"。

### 留存率（Retention Rate）
```
第 N 天留存率 = 注册第 N 天还活跃的用户 / 当天新增用户
```
| 留存 | 看什么 |
|------|--------|
| 次日 | 第一印象好不好 |
| 7 日 | 有没有形成使用习惯 |
| 30日 | 产品是否真正有价值 |

> **面试话术**：留存是产品的"体温计"。次留看拉新质量，30 留看产品价值。行业参考：电商次日留存 20-30% 算正常，30 日留存 8-15%。

### 复购率（Repurchase Rate）
```
复购率 = 购买 ≥ 2 次的用户数 / 总购买用户数
```
> 🏪 **店铺实况**：项目一中复购用户 **395 人**，复购率 **46.42%**，复购用户人均 2.5 单、人均消费 ¥414。老用户中人均购买 8.3 次（项目二），说明老用户忠诚度远高于大盘——这就是为什么项目二要聚焦存量运营。

> 复购率 = 用户忠诚度。我之前项目二的本质就是通过 RFM-G 分层找出复购潜力大的用户，做精准补贴。

### DAU / MAU / 用户粘性
```
DAU = 每天活跃用户数
MAU = 每月活跃用户数
粘性 = DAU / MAU
```
| DAU/MAU | 产品类型 |
|---------|---------|
| 10-20% | 工具型（携程、12306） |
| 20-30% | 内容型（知乎、小红书） |
| 30-50% | 社交型（抖音、微博） |
| 50%+ | 刚需社交（微信） |

### LTV（Life Time Value）客户终身价值
```
LTV = 平均客单价 × 平均购买次数 × 平均留存年限
```
> 🏪 **店铺实况**：如果老用户人均消费 ¥4,594、年均购买 3~4 次、留存 2~3 年，粗估 LTV ≈ ¥12,000~18,000。这时如果 CPA 是 ¥200，LTV/CAC ≈ 60~90，远高于 3 的健康线——说明拉新很划算，应该加大投放。

> 面试金句：**LTV / CAC > 3 才是健康的商业模式**（CAC = 获客成本）

## 七、RFM-G 指标体系（必考，你项目核心）

> 🏪 **项目二真实分层数据（N=11,997）：**

| 分层 | 人数 | 占比 | R(天) | F(次) | M(元) | G | 活动参与率 |
|------|------|------|--------|--------|--------|------|-----------|
| **高价值深耕用户** | 2,716 | 22.6% | 190 | 10.8 | 9,076 | 3.60 | 74% |
| **高潜唤醒用户** | 4,100 | 34.2% | 230 | 8.6 | 5,073 | 3.09 | 66% |
| **成长型用户** | 3,380 | 28.2% | 250 | 7.4 | 2,228 | 2.05 | 61% |
| **流失风险用户** | 1,801 | 15.0% | 305 | 5.6 | 1,184 | 1.18 | 50% |

> 🏪 **关键发现**：22.6% 的高价值用户贡献了远超其人口比例的 GMV；68.3% 的用户拓展到了服装品类；G 维度（品类拓展）与 R/F/M 的相关度约 0.3~0.5，说明捕获了传统 RFM 之外的增量信息。

### R（Recency）最近购买时间
```
R = 当前日期 - 用户最后一次购买日期（天）
```
| R 小（最近买过）| R 大（很久没买）|
|----------------|---------------|
| 活跃度高，流失概率低 | 可能已流失 |

> 🏪 **数据**：12,000 老用户平均 R = **238 天**，中位数 214 天。高价值用户 R 仅 190 天（~6 个月），流失风险用户 305 天（~10 个月没买）。
>
> **理论支撑**：近因效应 — 最近发生的行为对未来行为预测力最强。

### F（Frequency）购买频次
```
F = 用户在统计周期内的购买次数
```
| F 高 | F 低 |
|------|------|
| 忠诚用户，习惯已形成 | 新用户或低粘性用户 |

> 🏪 **数据**：人均 **8.3 次**，中位数 8 次。高价值用户 10.8 次 vs 流失风险用户 5.6 次。频次分布相对均匀（标准差 2.88），说明用户群整体粘性尚可。
>
> **理论支撑**：Ehrenberg NBD 模型 — 买过 3 次的人第 4 次购买概率远高于买过 1 次的人第 2 次。

### M（Monetary）消费金额
```
M = 用户在统计周期内的累计消费金额
```
| M 高 | M 低 |
|------|------|
| 高价值客户 | 低消费 |

> 🏪 **数据**：人均 **¥4,594**，中位数 ¥3,251。呈现明显的右偏分布——少数高消费用户拉高了均值。高价值用户 ¥9,076 vs 流失风险用户 ¥1,184，差距 7.7 倍。
>
> **理论支撑**：帕累托法则（二八定律）— 20% 用户贡献 80% 收入。

### G（Growth）品类拓展成长性——你的创新维度
```
G = 品类多样性 × (1 + 服装金额占比)
```
> 🏪 **数据**：G 均值 2.62，高价值用户 3.60，流失风险用户 1.18。**68.3% 的用户拓展到了服装品类**，这批用户的消费金额和频次都显著高于只买袜子的用户。
>
> **面试加分**：传统 RFM 只能回答"谁买了多少"，不能回答"用户是否在拓展消费品类"。G 维度对应的学术概念是 **Cross-Buying（交叉购买）**——Kumar et al. (2008) 在 *Journal of Marketing* 证明跨品类客户未来 CLV 显著更高。我再补充了一句"用乘法而非加法，让广度和深度产生交互效应"——这就能从"多加了一个指标"上升到"有方法论支撑的特征工程"。

### R、F、M 三者独立性
> **面试重点**：RFM 有效的根本原因——三个维度在统计上**相对独立**，分别捕获了流失概率（R）、行为惯性（F）、经济价值（M），合在一起覆盖客户价值的主要信息量。
>
> 🏪 在我的数据中，R 与 F 的相关系数约 0.3（弱相关），F 与 M 约 0.5（中等相关）——符合理论预期。如果两个维度高度共线（r > 0.8），就要考虑合并。

### RFM-G 分层常见切法
```
高价值:    R↑F↑M↑  → 重点维护，VIP 待遇     (我项目: 22.6%)
高潜唤醒:  R↑F↑M↗  → 品类拓展，套餐推荐     (我项目: 34.2%, 最大的群体)
成长型:    R↗F↘M↘  → 提升复购频次          (我项目: 28.2%)
流失风险:  R↓F↓M↓  → 大力度召回             (我项目: 15.0%)
```

## 八、AB 测试指标（面试高频）

> 🏪 **项目二 AB 测试真实数据**：我模拟了四种优惠策略对高潜唤醒用户的转化效果，每组 200 人。

### p 值（p-value）
```
p < 0.05 → 结果统计显著（不是随机波动造成的）
p ≥ 0.05 → 结果不显著（可能是运气）
```
> 🏪 **我的 AB 测试**：卡方检验 χ²=29.09, p=0.0000，说明四组间存在极显著差异。然后做两两比较——所有实验组 vs 对照组的 p 值都 < 0.01，全部显著。
>
> 面试话术：p 值只回答"有没有差异"，不回答"差异有多大"——所以还要看效应量（Lift）。

### 效应量（Lift / Effect Size）
```
Lift = (实验组转化率 - 对照组转化率) / 对照组转化率
```

> 🏪 **我的 AB 测试结果全表：**

| 组别 | 转化率 | 转化人数 | Lift | p 值 | 显著？ |
|------|--------|---------|------|------|--------|
| 对照组（无优惠） | 5.5% | 11/200 | — | — | — |
| A 组（10%折扣券） | 15.5% | 31/200 | **+181.8%** | 0.0011 | ✅ |
| B 组（满199减30） | 18.0% | 36/200 | **+227.3%** | 0.0001 | ✅ |
| C 组（买一送一） | 25.0% | 50/200 | **+354.5%** | <0.0001 | ✅ |

> 🏪 **业务解读**：C 组效果最好但成本也最高——实际决策中要算"增量毛利 vs 优惠成本"。如果满 199 减 30 的折扣成本远低于买一送一，B 组反而是更优选择。**AB 测试不只比转化率，最终要比 ROI。**

### 最小样本量
```
n = (Z² × p × (1-p)) / E²

Z = 1.96（95% 置信度）
p = 预估转化率（不知道就用 50%）
E = 期望误差（通常 1-5%）
```
> AB 测试前必须算样本量——样本太少，即使做出显著差异也靠不住。

### 常见坑
| 问题 | 说明 |
|------|------|
| **辛普森悖论** | 分组看都显著，合起来不显著（或反过来） |
| **多重比较** | 同时测很多组不用 Bonferroni 修正，假阳性 |
| **提前终止** | 看到显著就停，会高估效应量 |
| **新奇效应** | 用户短期对新变化有正向反应，长期回归均值 |

> 🏪 在我的 AB 测试中，我用卡方检验做**整体差异**判断，再用两两 z 检验做**具体对比**——这样避免了多重比较的假阳性问题。

## 十、面试高频追问 & 标准回答（含店铺实例）

### Q1: "GMV 涨了 20%，是好是坏？"
> **答**：不能只看 GMV 绝对值。要拆解——是 UV 涨了还是客单价涨了？如果 UV 大涨但转化率跌了，说明烧钱拉新但留存差。如果客单价涨了但订单数跌了，可能砍掉了低频用户。好增长 = GMV ↑ + CVR ↑ + AOV ↑。
>
> 🏪 **结合店铺**：比如我在项目二里模拟的 GMV 恢复——老用户月度 GMV 占比从 7 月的 32% 持续下滑到 12 月的 23%，经过精准分层运营后从底部开始回升到 34%。这个过程中 UV 没大涨（不是烧钱拉新），但高潜用户的转化率和客单价上去了——这才是健康增长。

### Q2: "PV 和 UV 的比值突然从 5 变成 15，你怎么看？"
> **答**：先排查是不是数据口径问题（埋点重复上报？爬虫流量？），再分析用户行为——是活动页吸引力强导致多浏览，还是页面卡顿导致反复刷新，还是刷量作弊。
>
> 🏪 **结合店铺**：我们的 PV/UV ≈ 1.9，比较低——但这是 DTC 单品牌的正常现象，SKU 少、品类窄。如果突变成 15，我会第一时间查是否有爬虫或埋点 bug。

### Q3: "转化率从 3% 跌到 1.5%，怎么排查？"
> **答**：漏斗逐层拆解→ 加购率跌了（商品/价格问题）、下单率跌了（竞品促销/运费变化）、支付率跌了（支付系统故障）→ 再按渠道/时段/用户群切片，定位是全局问题还是局部问题。
>
> 🏪 **结合店铺**：比如我们抖音渠道的转化率突然跌了——可能是因为竞品在抖音做了大促。我会拉出该渠道的漏斗数据，如果发现"加购→下单"环节暴跌而其他环节正常，就确认是价格竞争力问题。

### Q4: "ROI = 5 的渠道和 ROI = 2 的渠道，钱应该全投到 ROI=5 的吗？"
> **答**：不一定。ROI=5 可能已经是规模上限了（再投转化率会下降），ROI=2 可能是大渠道天花板高。要做**边际 ROI 分析**，看每多投 1 块钱带来的增量收入。还要考虑渠道的**获客质量**——新客 LTV 可能比这次 ROI 更重要。
>
> 🏪 **结合店铺**：我项目三中天猫 ROI 可能不如抖音好看，但天猫用户复购率可能更高（LTV 更大），抖音可能更多一次性冲动消费。所以我用 CPC × ROI 四象限同时看两个维度，而不是单看 ROI 一个数。

### Q5: "你怎么定义'活跃用户'？"
> **答**：要结合业务场景。电商通常定义为"近 30 天有浏览或购买行为"，内容产品可能是"近 7 天有阅读/互动"。定活跃窗口的核心依据是用户行为周期——买袜子的周期可能是 30-60 天，那就不能用 7 天窗口。
>
> 🏪 **结合店铺**：我们的袜子品类消费周期约 30-60 天，服装品类可能 60-90 天。所以我做 RFM 分层时 R 指标用"距上次购买天数"而非固定活跃窗口——这样更精准地识别不同消费节奏的用户。如果面试官追问"那为什么不统一用 30 天活跃窗口"，我会说"统一窗口会低估低频高价值用户（比如每 3 个月来买一次 ¥500 外套的人）。"

### Q6: "你的分析对业务有什么实际影响？"
> 🏪 **店铺标准回答**：
> - **项目一**：搭建了日常运营看板，让管理层每天 5 分钟就能看到 PV/UV/GMV 趋势和漏斗转化情况
> - **项目二**：通过 RFM-G 分层把 12,000 老用户分成 4 层，针对 34.2% 的高潜唤醒用户推品类拓展券，AB 测试证明买一送一可提转化率 354%
> - **项目三**：用四象限分析重新分配四个平台的广告预算，同时用全域归因纠正了末次点击模型对抖音的高估
> - **项目四**：150 个产品分级后，对 10.7% 的一般款提前预警、对不同品类按渠道制定差异化清仓策略
