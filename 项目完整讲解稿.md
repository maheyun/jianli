## 1. 开场：项目背景与个人定位

### ⭐ 项目背景

> "我实习的公司叫**跃动体育**，是一家 100 人规模的 DTC 运动服饰品牌。主要在天猫、京东、得物、抖音四个平台卖袜子、T恤、裤子、外套四个品类。实习期间我负责搭建了整个数据分析体系，覆盖了日常运营、用户分析、渠道优化、产品管理四个方向。"

**公司画像快速过一遍：**

| 维度 | 情况 |
|------|------|
| 规模 | ~100 人，DTC 模式（自产自销，无经销商） |
| 渠道 | 天猫 / 京东 / 得物 / 抖音 |
| 品类 | 袜子 / T恤 / 裤子 / 外套 四个大类 |
| 数据量 | ~20 万用户、500 SKU、365 天运营数据 |
| 技术栈 | Python + SQL + Jupyter + Dash + Power BI |

**为什么要强调这些？** 因为它决定了你的分析框架不是凭空想的——100 人公司的数据量级、四个真实渠道、四个有价格梯度的品类，这些约束条件让每一个分析都有据可依。

### ⭐ 数据来源与工具生态

> "我们公司的数据基础设施有三层。第一层是**标配工具**：聚水潭 ERP 做全平台订单管理和库存，没这个四个渠道的客服和仓储根本忙不过来。第二层是**各平台后台**：天猫用生意参谋、京东用京东商智、抖音用电 商罗盘、得物用商家后台——每个运营日常看的就是这些。第三层是**广告后台**：千川、京准通、引力魔方，看投放数据。"

**100 人规模公司的真实数据工具配置：**

| 层级 | 工具 | 用途 | 费用 |
|------|------|------|------|
| 标配 | ERP（聚水潭/旺店通） | 全平台订单汇聚 + 库存 + 发货 | ~1-3 万/年 |
| 标配 | 平台后台（生意参谋/商智/罗盘） | 单平台日常流量/转化/商品数据 | 部分免费 |
| 标配 | 广告后台（千川/京准通） | 投放效果数据 | 0（投广告就有） |
| 进阶 | 生意参谋·市场洞察专业版 | 天猫行业大盘 + 竞品数据 | ~900-1800 元/月 |
| 进阶 | 蝉妈妈 | 抖音同行达人数据 + 爆款分析 | ~300-500 元/月 |
| 自建 | **我的分析体系（Python+SQL+BI）** | 跨平台整合 + 自定义模型 + 看板 | 人力 |

> "我入职后发现的问题是：数据散落在四个平台后台和 ERP 里，老板要看跨平台对比时，需要人工从每个后台导出 Excel 再拼表——耗时长、口径不统一（天猫销售额含运费、抖音认付款成功、京东认确认收货）。**我做的事情不是重复平台已有的功能，而是把 ERP 的订单数据、各平台后台的流量/广告数据整合起来，做平台不会替你做的分析——跨平台 ROI 对比、自定义 RFM-G 分层、边际收益曲线、AB 测试框架。**"

> "可以这样理解：ERP 和各平台后台是**面粉**（原始数据），我做的是把面粉做成**面包**（分析洞察和策略建议）。面粉在哪都能买到，但面包的口味取决于师傅的手艺。"

### ⭐ 个人定位（实习生的真实角色）

> "首先要诚实地说——我来之前公司不是从零开始的。一个运营了一年多的 100 人电商公司，ERP 管着全平台订单，各平台后台（生意参谋/商智/罗盘）管着日常运营，运营团队每天手动拉数填 Excel 日报。**数据是有的，但三块短板很明显：**"

> "**第一，跨平台数据没有打通。** 四个渠道各自为政，天猫运营只看天猫、京东运营只看京东，老板要看全貌得等半天人工拼表。**第二，指标口径不统一。** 天猫销售额含运费、抖音认付款成功、京东认确认收货，没人做过统一校准。**第三，只有描述没有诊断。** 平台后台告诉你'昨天卖了多少'，但没人算过'抖音的边际 ROI 是不是在递减'、'复购率下滑到底是哪个用户层在流失'、'滞销品应该打折还是直接清仓'。"

> "所以我的定位不是推翻重来——**而是在公司已有的 ERP 和平台后台基础上，往上长出一层分析能力。** 用 Python 做 ETL 统一四个平台的数据口径，在 ERP 的订单数据上建了 RFM-G 分层模型和 AB 测试框架，用 Dash 和 Power BI 把 Excel 手工日报变成了自动化看板。地基是现成的，我做的是在毛坯房上做精装修。"

> "下面我从四个项目的递进关系来展开。"

## 2. 项目一：公司日常运营指标分析 ⭐

### 2.1 业务问题

> "刚入职时，公司虽然有 ERP 和各平台后台，但数据散落在四个地方，没有跨平台的统一视角。老板问我'店铺最近怎么样'，运营需要从四个后台分别拉数据、手动拼 Excel——半天就过去了。所以我做的第一件事是：**基于现有数据源，搭建一套跨平台、自动化的日常运营指标体系**。"

核心要回答四个问题：
1. 每天有多少人进来？（流量）
2. 进来的人有多少买了？（转化）
3. 买了的人还会再买吗？（复购）
4. 不同价值的客户怎么分布？（分层）

### 2.2 数据源与处理

> "数据来自公司的业务数据库，核心两张表：`user_behavior`（用户行为日志）和 `orders`（订单表）。我用 SQLite 本地模拟了 ~600 万条行为记录和 ~33 万条订单。"

**关键处理步骤：**

- **数据清洗**：时间格式统一、订单状态筛选（只保留 paid/shipped/completed 为有效订单）
- **日聚合**：`groupby(date)` 计算每日 PV/UV/付费用户数/GMV
- **漏斗构建**：用 `pivot_table` 构建用户-行为矩阵，统计每步触达的用户数

### 2.3 核心指标与发现

**① 流量分析（PV/UV）：**

> "日均 PV 约 900，UV 约 470，PV/UV 比约 1.9——说明用户平均每次访问看 2 个页面，黏性偏低，有提升空间。"

**② 转化漏斗：**

> "从浏览→加购→下单→支付，整体转化率约 5%。最大的流失在「浏览→加购」这一步，超过 60% 的用户浏览后没有加购行为。这说明商品详情页或定价策略可能有问题。"

**③ 复购分析：**

> "复购率约 46%，将近一半的客户会买第二次以上。对于 100 人的品牌来说这个数字相当不错，说明产品力在线。"

**④ RFM 分层：**

> "我用了经典的 RFM 模型：R（最近购买时间）、F（购买频次）、M（消费金额），用 `pd.qcut` 五等分打分，然后用 `np.select` 向量化分四层。"

分层结果与运营策略：

| 分层 | 占比 | 策略 |
|------|------|------|
| 高价值客户 | ~15% | 私域沉淀、新品优先体验 |
| 中高价值客户 | ~25% | 交叉销售、满减券促升级 |
| 中价值客户 | ~35% | 促销触达、低门槛券 |
| 低价值客户 | ~25% | 控制补贴、观察是否自然转化 |

### 2.4 技术亮点

> "这里遇到一个技术坑：`pd.qcut` 在数据有大量重复值时会报 `ValueError`。我封装了一个 `safe_qcut` 函数，自动降级为等距分箱 `pd.cut`，保证程序不中断。另外用 `np.select` 代替 `.apply(axis=1)` 做分层判断，速度提升 10-50 倍。"

```python
def safe_qcut(series, q, labels):
    """安全分箱：处理 pd.qcut 重复边界值问题"""
    try:
        result = pd.qcut(series, q, labels=labels, duplicates="drop")
    except ValueError:
        result = pd.cut(series, bins=min(q, len(series.unique())),
                        labels=labels[:min(q, len(series.unique()))])
    return result
```

> "这两个优化体现了**工程思维**——不是功能能跑就行，而是考虑代码的健壮性和运行效率。"

## 3. 项目二：老用户激活与价值提升 ⭐

### 3.1 业务问题

> "项目一做完后，我发现一个更深的问题：**老用户的 GMV 占比在持续下滑**。公司一直在拉新，但存量用户的价值没有被充分挖掘。于是有了项目二：从'粗放拉新'转向'精细存量运营'。"

### 3.2 数据建模：RFM-G

> "传统 RFM 只能描述'买得多不多、最近买没买'，但我们是多品类品牌——用户买袜子还是买外套，行为价值完全不同。所以我加入了第五个维度 **G（Growth，品类拓展潜力）**。"

**G 的定义：**

```
G = 品类多样性 × (1 + 服装品类金额占比)
```

> "逻辑是：一个用户尝试的品类越多，且在高客单价品类（服装）的消费占比越高，说明他的消费升级潜力越大。这个 G 值帮助我们识别'从配件往服装升级'的用户。"

**五表 JOIN 构建用户宽表：**

```
users → orders → order_items → products → user_activities
```

> "这个五表联查是整个项目最复杂的 SQL。我从 users 表出发，左连接订单，再连订单明细，再连产品信息，最后关联用户活动数据，构建出一张完整的用户行为宽表。"

### 3.3 用户分层

> "在 R/F/M/G 四个维度上分别 `qcut` 五等分打分（1-5 分），总分 4-20 分，用 `np.select` 分四层："

| 分层 | 条件 | 占比 | 运营策略 |
|------|------|------|---------|
| 高价值深耕用户 | ≥16 分 | 22.5% | 不打扰，私域重点维护 |
| 高潜唤醒用户 | 12-15 分 | 33.7% | 品类拓展券（从袜子→服装） |
| 成长型用户 | 8-11 分 | 29.1% | 阶梯优惠（买越多越便宜） |
| 流失风险用户 | <8 分 | 14.6% | 大力度召回（无门槛券 + 短信） |

### 3.4 AB 测试 ⭐

> "分层只是第一步，策略是否有效需要验证。我设计了 AB 测试来评估不同优惠工具的效果。"

**实验设计：**

| 组别 | 策略 | 样本量 |
|------|------|--------|
| 对照组 | 不干预 | 3,000 |
| 实验组 A | 10% 全场折扣 | 3,000 |
| 实验组 B | 199-30 满减券 | 3,000 |
| 实验组 C | 买一送一 | 3,000 |

**统计方法：**

> "我用**卡方检验**检验各组转化率是否有显著差异，用**双样本比例 z 检验**对两两比较进行验证。结果发现实验组 C（买一送一）的转化率提升最高，达到 +354.5%，p < 0.001，非常显著。但需要注意——买一送一的成本也最高，如果算上毛利率，满减券的 ROI 可能更好。"

**量化模拟：**

> "我还做了一个 GMV 恢复模拟：假设分层策略全部落地，按各分层的响应率估算，预计 GMV 可恢复约 15-20%。"

### 3.5 技术亮点

> "RFM-G 的核心创新在于**第五维度 G 的业务定义**——它不是套公式，而是从业务逻辑出发：多品类 DTC 品牌，用户价值不只体现在花多少钱，还在于**品类拓展潜力**。这个指标的提出体现了数据分析师的业务洞察力。"

## 4. 项目三：各平台 ROI 预算重新分配 ⭐

### 4.1 业务问题

> "公司四个平台都在投广告，但预算分配基本靠'老板拍脑袋'——哪个平台消耗快就加钱，ROI 高的反而没拿到预算。所以我用数据把这个问题算清楚：**每个平台到底值不值得投？钱应该怎么分？**"

### 4.2 六大效能指标

> "首先我算了每个平台的六大指标，形成标准化的效能画像："

| 维度 | 指标 | 天猫 | 京东 | 得物 | 抖音 |
|------|------|------|------|------|------|
| 转化效率 | CTR（点击率） | 2.1% | 1.8% | 3.5% | 4.2% |
| 转化质量 | CVR（转化率） | 8.3% | 6.5% | 12.1% | 5.8% |
| 成本效率 | CPC（单次点击成本） | ¥2.8 | ¥3.5 | ¥1.2 | ¥0.8 |
| 获客成本 | CPA（单次转化成本） | ¥33.7 | ¥53.8 | ¥9.9 | ¥13.8 |
| 回报效率 | ROI | 18.5× | 8.2× | 44.3× | 16.1× |
| 用户价值 | ARPU | ¥156 | ¥203 | ¥218 | ¥89 |

> "最反直觉的发现：**得物 ROI 高达 44 倍但预算只占 13%**，京东 ROI 只有 8 倍却占了 22% 的预算。资源严重错配。"

### 4.3 四象限分析

> "我把四个平台放在 **CPC × ROI** 的散点图上，用均值线切成四个象限："

```
         ROI ↑
             │
  低成本引流  │  高效拉新场
  (抖音?)    │  (得物 ✅)
             │
  ──────────┼──────────→ CPC
             │
   需优化    │  高价值种草地
  (京东 ⚠)  │  (天猫?)
```

> "得物落在'高效拉新场'象限（低成本 + 高回报），京东落在'需优化'象限（高成本 + 低回报）。这个可视化直接推动了后续的预算调整决策。"

### 4.4 边际 ROI 分析

> "光看平均 ROI 不够——投得越多，边际收益会递减。我用**二次多项式回归**拟合了每个平台的投入-产出曲线。"

**核心逻辑：**

```
revenue = a × spend² + b × spend + c    （二次回归拟合）
边际 revenue = 2a × spend + b
边际 ROI = 边际 revenue / 1（每多投 1 元的额外回报）
```

> "通过求导找到每个平台的边际 ROI 递减拐点，发现：**京东在预算超过 120 万后边际 ROI 已经降到 1.5×**，基本不赚钱了。而得物即使加到 25 万，边际 ROI 仍然有 8×。这意味着得物还有很大的加量空间。"

### 4.5 归因分析

> "同一个用户可能看了抖音广告、搜了天猫、最后在京东下单。转化功劳怎么分？"

> "我做了两种归因模型的对比：**末次点击归因**（功劳全给最后一个触点，简单但有偏）和**时间衰减归因**（按时间加权，越接近转化权重越大，更合理）。结果显示抖音在时间衰减模型下的贡献被严重低估——因为它更多是'种草'角色，在末次点击模型里被忽略了。"

### 4.6 预算优化

> "最后用**贪心算法**优化预算分配：每次选择当前边际 ROI 最高的渠道加 1 单位预算，直到总预算用完。目标函数综合了 ROI（60% 权重）和边际 ROI（40% 权重）。"

**优化建议：**

| 平台 | 优化前占比 | 优化后占比 | 变化 |
|------|-----------|-----------|------|
| 得物 | 13% | 28% | **+15%** ⬆ |
| 抖音 | 34% | 45% | **+11%** ⬆ |
| 天猫 | 31% | 16% | -15% ⬇ |
| 京东 | 22% | 10% | -12% ⬇ |

> "预估全域 ROI 可提升 **~15.6%**。这个结论不是拍脑袋的，是数据驱动的。"

### 4.7 技术亮点

> "多项式回归做边际分析是一个典型的'经济学 × 数据分析'跨界应用。边际递减是经济学常识，但用数据量化它是分析师的核心能力。另一个亮点是**双归因模型的对比**——不迷信单一模型，而是对比验证，体现批判性思维。"

## 5. 项目四：产品组合分析 ⭐

### 5.1 业务问题

> "前面三个项目解决了'卖多少、卖给谁、在哪卖'的问题，还差一个：**卖什么？**——哪些产品该主推、哪些该清仓、库存怎么管？"

### 5.2 健康度评分

> "产品的健康度不能只看销量——卖 1000 件但毛利只有 5% 的产品，不如卖 500 件但有 40% 毛利的产品。所以我用**双维度四象限**来评分："

```
毛利率 ↑
       │
 高毛利 │  潜力品        ★ 爆款 ★
 滞销品 │  (推广不足)    (卖得好+利润高)
       │
  ─────┼─────────────→ 售罄率
       │
       │  淘汰品        走量品
       │  (该清仓)      (薄利多销)
```

**分级标准：**

| 等级 | 条件 | 占比 | 策略 |
|------|------|------|------|
| 爆款 | 售罄率>55% 且 毛利率>35% | 8% | 补货优先、首页推荐 |
| 畅销款 | 售罄率>35% 且 毛利率>25% | 25% | 正常补货、关联推荐 |
| 一般款 | 售罄率>15% | 30% | 观察、适度促销 |
| 滞销款 | 其余 | 38% | 清仓打折、渠道分流 |

### 5.3 趋势预测

> "光看当前状态不够，需要知道产品在**走上坡路还是下坡路**。我用**线性回归**对每个产品的日销量时间序列拟合趋势线："

```
斜率 > 0 且显著 → 上升趋势 → 加码推广
斜率 ≈ 0        → 平稳     → 常规运营
斜率 < 0 且显著 → 下降趋势 → 分析原因/准备清仓
```

> "线性回归虽然简单，但足够判断趋势方向。对于 350 天的时间序列数据，它比移动平均更能捕捉长期趋势。"

### 5.4 智能补货模型

> "补货不能靠人工拍脑袋——SKU 多了根本盯不过来。我建了一个自动化补货模型："

```
补货点 = 预测日销量 × 提前期 + 安全库存
建议补货量 = max(0, 预测日销量 × (提前期 + 7天) - 当前库存)
```

> "核心是三个变量：**预测日销量**（从趋势模型来）、**提前期**（供应商交货时间）、**安全库存**（波动缓冲）。当当前库存 < 补货点时触发补货建议。"

### 5.5 清仓策略

> "滞销品不是一刀切打折——不同渠道的消化能力不同："

| 滞销类型 | 推荐渠道 | 原因 |
|---------|---------|------|
| 高客单价滞销 | 京东 | 用户消费力高、对折扣敏感度低 |
| 时尚款滞销 | 抖音 | 短视频展示效果好、冲动消费 |
| 基础款滞销 | 得物 | 年轻用户、对基础款价格敏感 |
| 大量库存滞销 | 天猫 | 流量最大、走量能力最强 |

### 5.6 技术亮点

> "产品组合分析的技术复杂度在于**多指标综合判断**——售罄率、毛利率、趋势斜率、库存周转，四个维度不是独立看的，而是要交叉分析。另外补货模型虽然公式简单，但 `reorder_point` 和 `safety_stock` 的参数校准需要业务经验——设太高了压资金，设太低了断货，平衡很关键。"

## 7. 常见追问 & 回答要点

### Q1：你的数据是模拟的，和真实数据有什么区别？

> "区别主要在三点：第一，真实数据有更多脏数据（空值、重复、格式错误），我在 `safe_qcut` 等函数中做了防御性处理来模拟；第二，真实数据有外部因素干扰（促销、节假日、竞品动作），我在分析中标注了这些潜在混淆因素；第三，真实数据的体量和复杂度更大，但分析方法论是通用的——SQL + pandas + 统计学，给 20 万条和 2000 万条的方法是一样的。"

### Q2：为什么选 RFM 而不是其他分层模型？

> "RFM 是电商最经典的模型，三个维度直观可解释，业务方容易理解。进阶的 RFM-G 是我根据业务特点做的改进——多品类品牌需要关注品类拓展潜力。如果资源允许，还可以用 K-Means 聚类做无监督分层，但可解释性会差一些。在业务场景里，**可解释性比模型复杂度更重要**。"

### Q3：AB 测试中买一送一效果最好，为什么不直接全上？

> "因为成本。转化率提升 354% 听起来夸张，但买一送一的毛利率可能直接打穿了。我强调了需要结合毛利率综合评估——这个思维很重要，数据分析师不能只看转化不看利润。实际上满减券（实验组 B）虽然转化率提升只有 ~120%，但毛利损失小，ROI 可能更高。"

### Q4：归因为什么不用数据驱动的模型（如 Shapley 值）？

> "Shapley 值确实更严谨——它计算每个渠道在所有可能组合中的边际贡献，是合作博弈论的标准解法。但它的计算复杂度是 O(2^n)，4 个渠道还好，如果 10 个渠道就炸了。在这个场景里，时间衰减模型已经足够好，且在业务可解释性上更优——'离转化越近权重越大'比'Shapley 值'更容易跟老板解释。"

### Q5：线性回归做销量预测会不会太简单了？

> "对于趋势判断（上升/平稳/下降），线性回归足够了。如果要做精确的日销量预测，确实可以用 ARIMA 或 Prophet。但这里要分清楚需求：**我只需要分类（三档），不需要精确值**。用简单的模型解决够用的问题，是工程思维。过度设计反而增加维护成本。"

### Q6：你做的东西和 BI 工程师有什么区别？

> "BI 工程师侧重于**数据工程**——ETL、数据仓库、看板搭建。我更侧重于**分析洞察**——从数据里找到业务问题、提出可落地的策略建议。当然边界不是绝对的，我也做了看板（Dash + Power BI），但我的核心价值在于'这个数说明了什么、该怎么办'，而不是'怎么把这个数展示出来'。"

### Q7：各平台后台也能做这些分析，你做的不就是重复工作吗？ ⭐⭐⭐

> "这是个很好的问题。我分三层来回答。"

> "**第一层，平台后台的确在快速进化。** 2025 年京东商智已经内置了 RFM 分层和流失预警，抖音罗盘打通了千川数据可以单页看投产比，生意参谋的竞争 Agent 能自动生成竞品报告。我也必须承认，如果只看**单平台内的描述性指标**（今天卖了多少、转化率多少、复购率多少），平台后台确实覆盖了大部分。"

> "**第二层，但平台有三重不可逾越的边界。** 第一重是**数据孤岛**——天猫看不到京东的数据，抖音看不到得物的数据。我项目三里发现'得物 ROI 44 倍但预算只占 13%'——这个洞察任何单平台后台都给不了你，因为没有任何一个平台会帮你做跨平台对比。第二重是**分析深度**——平台告诉你'复购率 46%'，我告诉你'买一送一 vs 满减券哪个效果更好，p 值多少，GMV 预估恢复 15-20%'。第三重是**模型定制**——生意参谋用的是 AIPL，不是标准 RFM；京东商智的 RFM 是通用版，不会替你设计'品类拓展潜力 G 值'。每一个品牌的业务特点不同，通用的分层模型不可能适配所有人。"

> "**第三层，2025 年一个有意思的趋势是——竞品数据反而越来越看不到了。** 京东把竞品数据改成了区间（'1千-3千'），天猫一直是模糊指数，第三方工具做'指数还原'已经被判赔 3000 万永久下架了。平台在收紧数据开放度。这意味着，就算你只做单平台分析，未来能依赖平台自带工具做的事情也会越来越少。**自定义分析的能力反而变得更加不可替代。**"

> "所以总结：平台后台是**传感器**，告诉你发生了什么；我做的是**诊断仪**，告诉你为什么、怎么办。两者是互补关系，不是替代关系。"

### Q8：你的跨平台数据从哪来的？平台之间不是不通吗？

> "首先要区分两种数据：**自己公司的数据**和**同行竞品的数据**。"

> "自己公司在各平台的订单和用户数据，**完全可以拿到**。我们公司用了聚水潭 ERP，天猫、京东、得物、抖音四个渠道的订单会自动汇集到 ERP 数据库——这是 100 人规模电商公司的标配。同时各平台后台（生意参谋、京东商智、抖音罗盘）支持导出自己店铺的流量和用户行为报表。广告数据从千川、京准通、引力魔方的后台导出。我把这些数据通过 Python + SQL 做了 ETL 清洗入库，统一了四个平台不同的指标口径——比如天猫销售额含运费、抖音认付款成功、京东认确认收货——然后在这个统一口径的基础上做跨平台分析。"

> "同行竞品的跨平台整合数据，**确实拿不到**——这是平台的数据壁垒。但我的分析不需要竞品的精确数字：项目三是优化**自己**的预算分配（哪里的边际 ROI 更高），项目二是做**自己**用户的 RFM-G 分层，项目四是对**自己**产品的健康度评分。这些都是用自身数据就能完成的内部优化，不依赖竞品数据。"

### Q9：那怎么看同行数据？

> "获取同行数据有六个主要渠道，按精度和成本排："

> "**第一，平台官方后台的竞品功能。** 生意参谋专业版（~900-1800 元/月）可以看竞品指数化销量、流量结构、引流词。京东商智可以看到竞品的区间化成交数据。抖音罗盘的竞争对比功能 2025 年已经免费开放了。但每条路都有瓶颈：天猫给的是模糊指数不是真实数字，京东已经改成了区间（'1千-3千'），而且每个平台都只能看自己平台内的。"

> "**第二，第三方全域数据平台。** 任拓（Nint）、魔镜洞察、蝉妈妈这类工具通过合法采购脱敏数据，能看到接近真实的竞品成交数据。任拓服务了超 80% 的世界 500 强消费品企业。不过价格是企业级的，小公司不一定买。"

> "**第三，行业报告。** 魔镜洞察每年发消费趋势白皮书（免费），艾瑞和易观有付费报告。但报告是行业级别的宏观数据，不是竞品级别的精细数据。"

> "**第四，公开数据采集。** 用八爪鱼这类工具合法采集商品页面上的价格、销量排名、评价等公开信息。只能拿到表面的、非敏感的信息。"

> "**第五，供应链反向调研。** 去 1688 搜同款找代工厂，看起订量反推竞品规模；看竞品招聘信息反推团队规模和方向。这些是间接推断，精度有限。"

> "**第六，消费者端。** 分析竞品的电商评价（差评就是竞品的弱点）、做问卷调研、社交媒体监听。"

> "2025 年的一条红线必须提：以前第三方工具能做的'指数还原真实销量'，已经被南京中院判赔 3000 万，功能永久下架。这意味着未来能看到的竞品数据只会越来越模糊。**依赖平台自带功能能做的事，会越来越少。**"

### Q10：ERP（聚水潭）不是已经有报表了吗？你的分析和它有什么区别？

> "聚水潭的报表是**运营层面**的——今天发了多少单、库存剩多少、退货率多少。它是管订单的，不是做分析决策的。ERP 不会告诉你：用户的复购率和品类拓展潜力有什么关系（RFM-G）、四个平台哪个的边际 ROI 开始递减了、买一送一和满减券哪个策略效果更好（AB 测试+统计检验）、滞销品应该清到哪个渠道亏损最小。这些需要**分析方法论和业务判断**，ERP 不会替任何人回答。ERP 是面粉，我做的是面包。"
